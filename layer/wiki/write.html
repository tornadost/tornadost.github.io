📝 Publishing a new post

  1. Add the cover image
    Save your banner or thumbnail as N.png inside
    /posts/assets/cover/. Make sure the filename exactly matches the post ID (e.g. 3.png).
  2. Create posts/feed/N/N.json
    It needs at least these fields: 
    {
  "title":   "My Awesome Idea",
  "excerpt": "A single-line teaser for the card.",
  "date":    "2025-05-29",
  "read":    4,
  "tags":    ["tag1","tag2"]
}
    This JSON powers both your homepage card and the full‐post page.
  3. Duplicate the HTML template
    Copy posts/feed/1/1.html → posts/feed/N/N.html.
    You don’t need to edit inside—the script auto-detects N from the filename.
  4. (Optional) Use Markdown
    If you prefer writing in Markdown, add posts/feed/N/N.md and set in your JSON: 
    {
  "markdown": true
}
    The loader will convert it on-the-fly via marked.js.
  5. (Optional) Update inline JSON
    If you’re still using the old inline array, add to your index.html’s <script id="post-data"> array: 
    {
  "id":      N,
  "title":   "My Awesome Idea",
  "excerpt": "A single-line teaser.",
  "img":     "posts/assets/cover/N.png",
  "date":    "2025-05-29",
  "read":    4,
  "tags":    ["tag1","tag2"]
}
  6. Commit & push
    Your home page will auto-fetch any /posts/feed/N/N.json, and /posts/feed/N/N.html will render the full article.

✍️ Writing the post content

In posts/feed/N/N.json you can include a full‐HTML body via: 

{
  "title":   "My Awesome Idea",
  "excerpt": "A single-line teaser.",
  "date":    "2025-05-29",
  "read":    4,
  "tags":    ["tag1","tag2"],
  "content": "<p>Welcome to my post! Here's the introduction.</p>\n<h2>Details</h2>\n<p>More text…</p>\n<ul><li>Point one</li><li>Point two</li></ul>"
}

Or if you set "markdown": true, write N.md instead: 

# My Awesome Idea

Welcome to my post! Here’s the introduction.

## Details

More text…

- Point one
- Point two

Your script will fetch the JSON and, if markdown is true, convert the MD file to HTML automatically.

Troubleshooting

  • Blank article – check that posts/feed/N/N.json is valid (no trailing commas) and that the MD/HTML body exists.
  • 404 cover image – filenames on GitHub Pages are case-sensitive; verify /posts/assets/cover/N.png exactly.
  • New post missing on grid – with auto-fetch you shouldn’t need to update index.html; otherwise, confirm the inline JSON entry’s id.

Need more help? See the GitHub Pages docs on JSON hosting and our repo README for advanced tips.
