📝 Publishing a new post

  1. Add the cover image
    Save your banner or thumbnail as N.png inside /posts/assets/cover/. Make sure the file name exactly matches the post ID (e.g. 3.png).
  2. Create posts/feed/N/N.json
    At minimum, include these fields: 
    {
  "title":   "My Awesome Idea",
  "excerpt": "A single-line teaser for the card.",
  "date":    "2025-05-29",
  "read":    4,
  "tags":    ["tag1","tag2"]
}
    This JSON powers both the grid card and the full-post loader.
  3. Duplicate the HTML template
    Copy posts/feed/1/1.html → posts/feed/N/N.html. You don’t need to edit inside—its script auto-detects N from the filename.
  4. (Optional) Use Markdown
    If you prefer writing in Markdown, add posts/feed/N/N.md and include: 
    {
  "markdown": true
}
    in your N.json. The page will convert it on-the-fly.
  5. (Optional) Update inline JSON
    Only if you’re not using the auto-fetch approach above, add an entry in index.html’s <script id="post-data"> array: 
    {
  "id":      N,
  "title":   "My Awesome Idea",
  "excerpt": "A single-line teaser.",
  "img":     "posts/assets/cover/N.png",
  "date":    "2025-05-29",
  "read":    4,
  "tags":    ["tag1","tag2"]
}
  6. Commit & push
    Your home page will detect the new JSON, and /posts/feed/N/N.html will render your content.

✍️ Writing the post content

Inside your posts/feed/N/N.json, you can supply the full HTML for your article using a content field. For example: 

{
  "title":   "My Awesome Idea",
  "excerpt": "A single-line teaser.",
  "date":    "2025-05-29",
  "read":    4,
  "tags":    ["tag1","tag2"],
  "content": "
Welcome to my post! Here’s the introduction.
\n
Section heading
\n
More text…
\n
  • Point one
  • Point two
"
}

Or, if you set "markdown": true, you can write posts/feed/N/N.md: 

# My Awesome Idea

Welcome to my post! Here’s the introduction.

## Section heading

More text…

- Point one
- Point two

The loader will convert that Markdown to HTML automatically. This approach keeps your JSON clean and lets you write in Markdown syntax.

Troubleshooting

  • Blank article – ensure your JSON is valid (no trailing commas) and that the content or markdown field points to existing HTML/MD.
  • 404 cover image – file names on GitHub Pages are case-sensitive; verify /posts/assets/cover/N.png matches exactly.
  • New post missing on grid – if you’re using auto-fetch, no action is needed; otherwise, confirm you added the object to the inline array with the correct id.

Need more help? Check out the GitHub Pages docs on JSON hosting, and our repository’s README for advanced tips.
