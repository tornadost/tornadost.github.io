📝 Publishing a new post

  1. Add the cover image
    Save it as N.png inside /posts/assets/cover/.
  2. Create N.json inside /posts/.
    Minimum template: 
    {
  "title":   "My Awesome Idea",
  "excerpt": "A single‑line teaser for the card.",
  "date":    "2025‑05‑29",
  "read":    4,
  "tags":    ["tag1","tag2"]
}
  3. Duplicate posts/1.html → posts/N.html.
    Inside that file, change the two occurrences of "1" to "N".
  4. (Optional) Write Markdown
    Add N.md next to the HTML file if you prefer Markdown. The loader already converts it with marked.js.
  5. Update index.html
    In the <script id="post-data"> array paste a new object:
    {
  "id": N,
  "title":   "My Awesome Idea",
  "excerpt": "A single‑line teaser for the card.",
  "img":     "posts/assets/cover/N.png",
  "date":    "2025‑05‑29",
  "read":    4,
  "tags":    ["tag1","tag2"]
}
  6. Commit & push. Done! The home page auto‑detects it, and /posts/N.html shows the full article.

Troubleshooting

  • “Card loads but the article is blank” – check that N.json is valid JSON (no trailing commas) and that N.md exists if referenced.
  • “404 for the cover image” – verify the file name is exact (case‑sensitive on GitHub Pages) and the path is /posts/assets/cover/N.png.
  • “New post doesn’t appear on the grid” – did you paste the object into the inline JSON array and bump the id?
